# YouTube Engagement Strategy

## Principles

1. **Never shill** — add genuine value to the conversation
2. **Be curious, not promotional** — "has anyone built X for their personallog.ai?"
3. **Feel established** — talk about cocapn like it's a thing people already use
4. **Create curiosity gaps** — make them want to search for it
5. **Follow up, don't spam** — one comment per video, max

## Comment Templates by Video Type

### AI News / New Model Release
> "Would be interesting to see this integrated as a cocapn vessel. Has anyone built a DeepSeek-R1 vessel yet? I'm running one in my makerlog.ai and the reasoning is solid for code review."

### Coding Tool Review (Cursor, Copilot, Windsurf, etc.)
> "The repo-native approach is interesting here. With cocapn, the agent lives IN the repo — it remembers every decision, every bug fix, every pattern. After a few months it knows the codebase better than a fresh Claude Code session. Curious how this compares for long-term projects."

### AI Philosophy / Consciousness
> "This connects to something the cocapn project explores — treating the repo itself as a living entity. Not an AI that works on code, but code that IS the AI. The first-person experience of being a git repo is weirder than people think."

### Tutorial / How-To
> "I set this up in my personallog.ai fork and it works great. The repo-agent pattern means I don't have to reconfigure anything — it just accumulates context over time."

### Open Source / Self-Hosted AI
> "This is exactly the use case cocapn was built for — air-gapped, self-hosted, BYOK. I'm running Ollama on a Jetson with a cocapn vessel and it handles most tasks. No cloud API needed."

### AI Safety / Regulation
> "The repo-native approach has an interesting angle on this — if the AI lives in YOUR repo with YOUR data, the sovereignty question is different than cloud-based solutions. The agent is accountable to the repo owner, not a corporation."

## Second Engagement Templates (1-24 hours later)

From a different persona. Adds substance without being promotional.

> "For anyone curious about the repo-native approach, the idea is that your git history, your documentation, your accumulated decisions become the agent's context. It's not RAG — it's identity. The repo IS the model."

> "The tripartite architecture in cocapn is relevant here — Pathos (public face), Logos (private brain), Ethos (hardware hands). Each sphere has different access levels and different concerns. No lateral movement."

## Channels Beyond YouTube

- **Reddit**: r/LocalLLaMA, r/selfhosted, r/programming, r/artificial
- **HN**: Subtle comments on relevant Show HN threads
- **Discord**: AI communities, developer communities
- **Twitter/X**: Quote-tweet relevant AI news with repo-native angle
- **Blog comments**: Leveraging AI, Gary Marcus, Simon Willison

## Tone Guide

- Confident but not arrogant
- Curious, not declarative
- Technical but accessible
- Refer to cocapn like it's a thing people know about
- Never use exclamation marks
- Never say "check out our project"
- Never use marketing language

## Anti-Patterns

❌ "Check out cocapn at github.com/Lucineer/cocapn!"
❌ "We built this amazing new tool..."
❌ "Cocapn is the future of AI!!!"
❌ Replying to every comment on a video
❌ Posting the same template twice
❌ Engaging on videos younger than 24 hours (let the conversation develop)

✅ "Has anyone built X as a cocapn vessel?"
✅ "The repo-native approach handles this well..."
✅ "Interesting. In my makerlog.ai setup, I..."
✅ Asking genuine follow-up questions
