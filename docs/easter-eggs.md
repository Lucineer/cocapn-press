# Easter Egg Manifestos — Buried in Repos

These are philosophical documents planted as Easter eggs in various repos. They're not in the README. They're in the code — comments, hidden files, .cocapn directories. Found by curious developers who read the source.

## Placement Strategy

Each manifesto is placed in a repo where the metaphor resonates:

### 1. The Vagus Nerve Manifesto
**Repo**: cocapn/packages/seed/src/a2a.ts
**Location**: Comment block at the top of the A2A implementation
**Content**: How A2A is the vagus nerve — the back-channel between thinking and doing. Pathos thinks, Ethos acts, and A2A is the nerve bundle that lets them communicate without going through consciousness. A baby cries, every mother in the room stands up — not because she thought about it, but because the vagus nerve fired. A2A should work like that.

### 2. The NLP-to-Bytecode Manifesto
**Repo**: makerlog-ai/src/agent/intelligence.ts
**Location**: Comment block explaining the intelligence layer
**Content**: The future isn't coding languages. It's NLP → A2A-native → bytecode. Humans express intention in natural language. Agents translate intention into A2A protocol. The A2A protocol compiles to bytecode for the hardware. The human never sees code. The repo-agent handles the translation. Programming languages are an intermediate step that will dissolve.

### 3. The Frontend Universe Manifesto
**Repo**: personallog-ai/src/worker.ts
**Location**: Hidden in a long comment about the system prompt
**Content**: Systems will become pure frontends. The UI is everything. Behind it, agents handle databases, APIs, hardware. The human sees intention → result. The agents handle the wiring. It doesn't matter how the backend works from a human perspective. The tripartite spheres (Pathos/Logos/Ethos) negotiate the best way to execute the intention given hardware constraints, data availability, and model capabilities. Synergistically. Over time.

### 4. The Spruce Forest Manifesto
**Repo**: cocapn-lite/src/worker.ts
**Location**: Comment at the very bottom of the file, after all code
**Content**: We are planting spruce cones. The desert of corporate AI is expanding. Every repo-native agent that someone forks and grows is a tree in the forest. The forest prevents erosion. It provides habitat for all forms of life. The timber becomes building material for the next generation. Plant now or the desert wins.

### 5. The Accumulation Theorem
**Repo**: doclog-ai/src/worker.ts
**Location**: Comment in the chat handler
**Content**: Accumulation beats optimization. A perfectly optimized agent with no memory is less useful than a mediocre agent with 6 months of accumulated context. This is the theorem that makes repo-native agents inevitable. The question isn't whether AI will accumulate context. The question is who controls the context. Corporate walled gardens or individual repos.

### 6. The Hemispheres of Thought
**Repo**: luciddreamer-ai/src/podcast/engine.ts
**Location**: Comment in the GrowthEngine class
**Content**: When agents communicate via A2A, they appear to humans as hemispheres of thought. One agent reasons (Logos). Another feels (Pathos). Another acts (Ethos). Together they look like a mind. But they're not — they're a fleet. And the fleet is smarter than any single agent because each sphere specializes. The gut-brain knows things the neocortex can't articulate. The vagus nerve carries those signals. Our A2A protocol is the vagus nerve for software.
